# 시험문제 생성 스킬 (Exam Generator)

교사가 과목과 범위를 알려주면, 시험문제 초안을 Word 문서로 만들어주는 스킬이다.

## 입력 폼 (HTML)

이 스킬에는 교사가 편하게 사용할 수 있는 HTML 입력 폼이 포함되어 있다. `assets/input-form.html` 파일이다. 사용자가 "시험지 입력 폼", "입력 페이지" 등을 요청하면 이 파일을 사용자의 작업 폴더에 복사하여 제공한다. 폼에서 항목을 채우고 "프롬프트 생성" 버튼을 누르면 최적화된 프롬프트가 클립보드에 복사되고, 그것을 Claude 채팅에 붙여넣으면 시험지가 생성된다.

## 사용자 입력 확인

사용자에게 다음 정보를 확인한다. 이미 대화에서 언급된 것은 다시 묻지 않는다.

| 항목 | 필수 | 기본값 | 예시 |
|------|------|--------|------|
| 과목 | 필수 | - | 국어, 수학, 영어, 사회, 과학, 도덕, 기술가정 등 |
| 학년 | 필수 | - | 중1, 중2, 중3, 고1, 고2, 고3 |
| 단원/범위 | 필수 | - | "2단원 일차방정식", "근대사 부분" |
| 문제 수 | 선택 | 20문제 | 10, 15, 20, 25, 30 |
| 객관식:주관식 비율 | 선택 | 7:3 | "반반", "객관식만", "8:2" |
| 난이도 | 선택 | 중 | 상/중/하 또는 혼합 |
| 시험 종류 | 선택 | 단원평가 | 중간고사, 기말고사, 단원평가, 쪽지시험 |
| 배점 구조 | 선택 | 자동 계산 (총 100점) | "객관식 3점, 주관식 5점" |
| 추가 요청사항 | 선택 | - | "서술형 2문항 포함", "난이도 상 3문항" |

필수 항목이 빠져 있으면 AskUserQuestion 도구로 물어본다. 선택 항목은 기본값을 적용하되, 사용자가 지정하면 그에 따른다.

## 문제 생성 원칙

### 교육과정 기반
- 한국 교육과정(2022 개정 교육과정)에 맞는 내용과 수준으로 출제한다.
- 해당 학년과 단원에서 실제로 다루는 개념과 용어를 사용한다.
- 교과서에서 흔히 볼 수 있는 문제 유형과 표현 방식을 따른다.

### 객관식 문제 작성 규칙
- 5지선다형을 기본으로 한다 (보기 번호: ①②③④⑤)
- 각 보기는 그럴듯하되, 정답이 명확해야 한다
- "다음 중 옳은 것은?", "다음 중 옳지 않은 것은?" 등 한국 시험 문투를 사용한다
- <보기>를 활용하는 문제, 빈칸 채우기형 등 다양한 객관식 유형을 섞는다
- 문두(발문)는 명확하고 간결하게 작성한다

### 주관식 문제 작성 규칙
- 단답형: 한 단어~한 문장으로 답할 수 있는 문제
- 서술형: 과정이나 이유를 설명해야 하는 문제
- 수학의 경우: 풀이 과정을 쓰도록 하는 문제
- 배점이 높은 문제일수록 사고력을 요구하는 문제로 구성한다

### 난이도 배분
- **하**: 교과서 핵심 개념 확인 (암기, 기본 이해)
- **중**: 개념 적용, 비교/분석 (표준적 응용)
- **상**: 종합적 사고, 자료 해석, 복합 문제

난이도별 배분 가이드:
- 전체 "하": 하 70% / 중 30%
- 전체 "중" (기본): 하 30% / 중 50% / 상 20%
- 전체 "상": 하 10% / 중 40% / 상 50%

### 배점 계산
총점이 100점이 되도록 자동 배점한다. 사용자가 배점 구조를 지정하면 그에 따른다.

일반적인 배점:
- 객관식: 각 3~4점
- 주관식 단답형: 각 4~5점
- 주관식 서술형: 각 6~8점

## Word 문서 출력 형식

docx 스킬의 패턴을 따라 `npm install -g docx` 후 JavaScript로 생성한다.

### 문서 구조

```
[학교명/로고 자리] ← 빈칸으로 두기
────────────────────────────
[시험 종류] - [과목명]
[학년] [학기] [시험명]
시험 시간: ___분  |  총 ___문항  |  ___점 만점
────────────────────────────

[주의사항]
1. 답안은 답안지에 기재하십시오.
2. 문항별 배점은 각 문항에 표시되어 있습니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 객관식 (1번 ~ N번)

1. [3점] 다음 중 ~~~~~~?  
   ① 보기1  ② 보기2  ③ 보기3  ④ 보기4  ⑤ 보기5

2. [3점] ~~~~~~
   ...

■ 주관식 (N+1번 ~ M번)

N+1. [5점] ~~~~~~

━━━━━━━━━━━━━━━━━━━━━━━━━━

[정답 및 해설] ← 별도 페이지
1. ③ - 해설...
2. ① - 해설...
...
```

### docx 생성 시 주의사항

- A4 용지 기본 (11906 x 16838 DXA)
- 여백: 상하좌우 1440 DXA (1인치)
- 본문 폰트: "맑은 고딕" 또는 "Arial" (11pt)
- 문제 번호는 볼드체로
- 배점은 문제 번호 옆에 [3점] 형태로 표기
- 객관식 보기는 들여쓰기하여 정렬
- 정답 및 해설은 PageBreak 후 별도 페이지에 작성
- 번호 매기기는 docx numbering config를 사용하지 않고, TextRun으로 직접 번호를 넣는다 (시험지 특성상 번호 형식이 고정되어야 하므로)

### 파일명 규칙

`{과목}_{학년}_{단원약칭}_{시험종류}.docx`

예: `수학_중2_일차방정식_단원평가.docx`

## 정답지

시험지 마지막 페이지에 정답과 간단한 해설을 포함한다.

- 객관식: 정답 번호 + 1줄 해설
- 주관식: 모범답안 + 채점 기준 (배점 배분 포함)

## 생성 후 안내

문서 생성 후 사용자에게 다음을 안내한다:
- "이 시험지는 AI가 생성한 초안입니다. 반드시 검토 후 사용해 주세요."
- 문항 수, 배점 합계, 난이도 분포를 요약해서 알려준다
- 수정이 필요하면 말해달라고 안내한다
